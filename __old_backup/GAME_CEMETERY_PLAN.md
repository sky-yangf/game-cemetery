## 一、产品定位

### 一句话
一个**有温度的游戏墓园**——玩家可以来悼念死去的游戏，点蜡烛、写留言、分享自己的游戏记忆。

### 跟竞品的差异
| 项目 | 萌娘 wiki 停服列表 | 数字墓园 |
|---|---|---|
| 形式 | 纯文字表格 | 视觉化墓园（黄昏/枯树/乌鸦/墓碑） |
| 情感 | 无 | 墓志铭 + 幽默评语 + 蜡烛 |
| 互动 | 无 | 点蜡烛/留言/分享截图/"我玩过" |
| 主题 | 无 | 3 种风格可切换 |
| 审核 | Wiki 编辑 | 用户提交 + 管理员审核 |
| 分类 | 无 | 按厂商/停运时间/死因筛选 |

## 三、3 种墓园风格

### 主题系统设计
用 CSS 变量 `[data-theme="xxx"]` + `localStorage` 记忆。

| 主题 | 键名 | 天空 | 墓碑石材 | 氛围 |
|---|---|---|---|---|
| **暮色** | `dusk` | 暖橙→深紫 | 灰石/砂岩 | 黄昏墓园（默认） |
| **暗夜** | `night` | 深蓝→纯黑 | 暗花岗岩 | 哥特暗夜 |
| **淡雅** | `dawn` | 浅灰→白 | 白大理石 | 白昼追思 |

### 切换方式
右上角 3 个小图标按钮：
```
[🌅 暮色] [🌙 暗夜] [☀️ 淡雅]
```
点击切换，`localStorage` 记忆选择。

## 六、数据模型

### games（游戏墓碑）
```
id              INTEGER PRIMARY KEY
name            TEXT    游戏名
icon            TEXT    emoji/图标
publisher       TEXT    发行商
release_date    TEXT    上线日期 (YYYY-MM)
death_date      TEXT    停运日期
death_reason    TEXT    死因
epitaph         TEXT    墓志铭（官方/玩家总结）
comment         TEXT    一句幽默评语
category        TEXT    分类标签（端游/手游/页游）
status          TEXT    审核状态：approved | pending | rejected
submitter_id    INTEGER 提交者用户ID（外键）
created_at      TEXT
```

### comments（留言）
```
id              INTEGER PRIMARY KEY
game_id         INTEGER 关联游戏（外键）
user_id         INTEGER 用户ID（外键）
content         TEXT    留言内容
image_url       TEXT    截图URL（可选）
status          TEXT    approved | pending | deleted
created_at      TEXT
```

### users（用户）
```
id              INTEGER PRIMARY KEY
email           TEXT    UNIQUE
password_hash   TEXT
name            TEXT    显示名
role            TEXT    user | admin
created_at      TEXT
```

### candles（蜡烛）
```
id              INTEGER PRIMARY KEY
game_id         INTEGER
user_id         INTEGER  NULL=匿名
created_at      TEXT
```

## 七、API 端点

### 公开（无需登录）
```
GET    /api/games                列表（?publisher=&death_year=&death_reason=&sort=）
GET    /api/games/{id}           详情（含蜡烛数、前10条留言）
GET    /api/candles/{game_id}    蜡烛数
POST   /api/candles/{game_id}    点蜡烛（可匿名）
```

### 需登录
```
GET    /api/games/{id}/comments  留言列表（?page=&sort=）
POST   /api/games/{id}/comments  发表留言（body: content, image?）
POST   /api/games                提交新墓碑（body: 游戏信息）
GET    /api/me                   当前用户信息
```

### 管理员
```
PUT    /api/games/{id}/approve   审核墓碑（body: status）
PUT    /api/comments/{id}/approve 审核留言
DELETE /api/comments/{id}        删除留言
```

### 鉴权
```
POST   /api/auth/register       注册（body: email, password, name）
POST   /api/auth/login          登录（返回 JWT token）
```

## 八、技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| **前端** | 纯 HTML/CSS/JS | 无框架、单文件部署、Cloudflare Pages 零配置 |
| **后端** | FastAPI + SQLite | 单文件启动、0 外部依赖 |
| **鉴权** | JWT (python-jose) + bcrypt | 复用 KnowBase 骨架 |
| **图片** | 后端 `static/uploads/` | SQLite 单机够用 |
| **部署** | Cloudflare Pages(前端) + Railway(后端) | 免费额度够用 |

## 九、部署方案

### 前端：Cloudflare Pages
```
1. 上传 game_cemetery/ 到 Git 仓库
2. Cloudflare Pages 绑定
3. 构建命令：无（纯静态）
4. 输出目录：./
5. HTTPS 自动 + 全球 CDN + 免费
```

### 后端：Railway
```
1. 上传 backend/ 到 Git
2. Railway 绑定
3. 启动命令：uvicorn app.main:app --host 0.0.0.0 --port $PORT
4. 免费额度 500h/月 + 1GB DB
```

### 前端指向后端
在 `main.js` 顶部一行：
```js
const API_BASE = 'https://your-app.railway.app/api';
// 本地开发时改为 http://127.0.0.1:8000/api
```
